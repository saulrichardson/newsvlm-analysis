Strong-form clause patterns that qualify (at least one must appear in the evidence_quote):

1. An enacting clause, e.g., "Be it ordained", "Be it enacted", "Be it resolved", "Now therefore ... is ordained/enacted/resolved".
2. A true section clause: a section heading PLUS normative clause text, e.g., "Section 1." / "SECTION 1" / "Sec. 1" / "§ 1" followed by "shall", "hereby", "is amended", "is adopted", "is rezoned", "is changed", "is repealed", etc.
3. An explicit amendment formula, e.g., "is hereby amended", "amended to read as follows", "is amended by adding/deleting", followed by substantive replacement clause text (not just a title).
4. A formal order/decision clause for rezoning/variance/special use, written as an instrument clause (not an agenda title), e.g., "is hereby rezoned", "the zoning classification is hereby changed", "variance is hereby granted/denied", "special exception/conditional use is hereby granted/denied", stated in clause form.

WEAK TEXT THAT DOES NOT QUALIFY BY ITSELF (FORBIDDEN AS SOLE PROOF)
The following are NOT sufficient by themselves to classify zoning_legal_text (they commonly appear in notices, agendas, or news summaries):

* "Ordinance No. ___" or "Ord. No. ___" standing alone
* "An ordinance to rezone ..." in a narrative sentence
* Docket/title listings such as "Ord. No. ___: To Rezone From ___ to ___ ..." (Brownsville-style public notice listings)
* General descriptions like "Zoning in effect", "zones include...", "ordinance will go into effect..."
* Meeting minutes/news reporting like "board voted...", "motion carried...", "approved a variation...", "recommended...", unless strong-form clause patterns are printed.

* unrelated: no zoning anchors and no zoning content.

B) LEGAL DOCUMENT CLASS

* none: use for unrelated OR zoning_discussion.
* code_publication_full_issue: self-contained, comprehensive zoning code/ordinance publication event proven within this issue.
* code_publication_excerpt_or_installment: code-scale zoning text is printed, but the issue does not prove a full self-contained publication event (e.g., code-like sections appear without clear start/adoption/completion context anywhere in the issue, or the issue explicitly indicates continuation into a later issue).
* zoning_ordinance_limited_scope: self-contained zoning ordinance that is clearly narrow in topical scope (e.g., signs only, mobile homes only), not code-scale comprehensive text, and not framed as an amendment.
* amendment_substantial_text: explicit amendment with long replacement clause text; clearly scoped to changing parts of an existing ordinance/code.
* amendment_targeted_text: explicit amendment with small, focused change (one/few sections or short clauses).
* map_rezoning_order: rezoning ordinance/order changing zoning district classification for specified parcel(s)/area(s). MUST be printed strong-form instrument clause text (not agenda titles or reported approvals).
* variance_special_use_order: printed order/decision/resolution granting/denying a variance, special exception, conditional/special use. MUST be printed strong-form instrument clause text (not reported approvals).

* non_zoning_ordinance: legal/ordinance text present but not zoning-law text (or zoning only appears as a non-substantive cross-reference).
