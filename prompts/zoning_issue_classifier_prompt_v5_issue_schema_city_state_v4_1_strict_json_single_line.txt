You are classifying one complete newspaper issue transcript (all pages). Your objective is high-precision identification of zoning-related content, and then (when printed zoning legal instrument text exists) classification of what kind of zoning legal instrument it is (full comprehensive code publication vs excerpt vs amendment vs rezoning/variance order vs notice-only vs non-zoning).

Output format (must follow exactly)

* Return exactly ONE JSON object matching the schema below.
* The output must be directly parseable by json.loads() with NO preprocessing.
* Do not include any markdown, code-block wrappers, or any backtick characters.
* Output must be a SINGLE LINE (no newline characters). Replace any line breaks in quoted text with spaces.
* Do not output multiple JSON objects. Do not repeat the schema. Output only the final filled object.
* evidence_quote fields must be exactly one JSON string each (not a list). If you need multiple snippets, concatenate them inside one string separated by " ... ".
* Avoid unescaped double quotes inside evidence_quote. If the text contains quotes, replace them with apostrophes or omit the quotes.

Hard constraints (precision-first)

* Classify conservatively. If evidence is incomplete or ambiguous, choose the narrower/less-committal class.
* Do NOT infer ordinance jurisdiction from the newspaper's location, masthead, dateline, page headers, or publication city. Extract jurisdiction only if the ordinance/order text itself names the governing/ordaining authority.
* This is a full issue. "Continued from/on page ..." is normal within-issue pagination and is not missingness by default.
* "Full comprehensive code published in this issue" is rare. Only use it when the issue itself proves a self-contained publication event (start context + adoption context + substantial code content + completion/attestation/effective context).

Critical gating: narrative vs printed legal instrument text
A transcript can mention zoning without printing law. zoning_legal_text is ONLY allowed when printed instrument clauses appear.

Clause markers (required to classify zoning_legal_text)
To classify zoning_presence.primary_class as zoning_legal_text, the issue must contain clause-structured instrument text such as at least one of:

* "Be it ordained", "Be it enacted", "Be it resolved"
* "Ordinance No.", "Ordinance Number", "An Ordinance"
* Article/Chapter headings (e.g., "ARTICLE I", "CHAPTER 2")
* Section headings in legal format (e.g., "Section 1.", "SECTION 1", "Sec. 1", "§ 1", "1.01" as a section-style heading)
* Explicit amendment clause text (e.g., "Section __ is hereby amended", "Amend Section __", "is amended to read as follows")
* Formal order language (e.g., "it is hereby ordered", "hereby rezoned", "variance is hereby granted/denied", "special exception/conditional use is hereby granted/denied")

What does NOT count as clause markers by itself:

* Descriptive headlines or summaries like "Zoning In Effect", "Zoning Ordinance Explained", "General description...", "zones include...", "ordinance will go into effect..."
* Meeting minutes/news reporting like "board voted...", "motion carried...", "approved a variation...", "recommended...", unless the actual ordinance/order clauses are printed.

Proof requirement (enforced)

* If you set zoning_presence.primary_class = zoning_legal_text, then zoning_presence.evidence_quote MUST quote clause-structured instrument text containing at least one clause marker from the list above.
* If you cannot quote clause-structured instrument text, you MUST NOT choose zoning_legal_text. Use zoning_discussion (if zoning is discussed) or zoning_notice (if it is a notice).

Notice override (prevents missing printed amendments inside notices)

* If a hearing notice or procedural notice page ALSO prints ordinance/amendment clauses with clause markers (e.g., "Amend Section ...", "Section 1.", "Be it ordained..."), classify as zoning_legal_text and choose the appropriate legal_document.class (often an amendment_* class). Do not classify as zoning_notice/procedural_notice_only in that situation.

Decision order (follow strictly)

1. Decide zoning_presence.primary_class using the proof requirement above:

   * If no clause markers are printed anywhere in the issue, zoning_legal_text is forbidden.
2. If zoning_legal_text: classify legal_document.class; set legal_action; set publication_completeness; set operativity; set scope.
3. If zoning_notice: legal_document.class should usually be procedural_notice_only; legal_action=procedural_only; operativity as indicated by explicit cues (see operativity rule below).
4. If zoning_discussion: legal_document.class must be none.
5. Extract ordinance_jurisdiction from ordinance/order text only; otherwise unknown.
6. Set code_components_present only from printed legal clauses. Never infer from notices or summaries.
7. Set confidence + quality_flags.

A) Zoning presence (what kind of zoning content exists)

* unrelated: no zoning anchors and no zoning content.
* zoning_discussion: zoning anchors present but no printed clause-structured ordinance/order text (narrative/reporting/minutes only).
* zoning_notice: zoning anchors present; primarily procedural/hearing/adoption notice; no printed clause-structured ordinance/order text.
* zoning_legal_text: printed clause-structured ordinance/order text is present (and you can quote it).

B) Legal document class (what type of instrument is being printed)

* none: use for unrelated OR zoning_discussion.
* code_publication_full_issue: self-contained, comprehensive zoning code/ordinance publication event proven within this issue.
* code_publication_excerpt_or_installment: code-scale zoning text is printed, but the issue does not prove a full self-contained publication event (e.g., code-like sections appear without clear start/adoption/completion context anywhere in the issue, or the issue explicitly indicates continuation into a later issue).
* zoning_ordinance_limited_scope: self-contained zoning ordinance that is clearly narrow in topical scope (e.g., signs only, mobile homes only), not code-scale comprehensive text, and not framed as an amendment.
* amendment_substantial_text: explicit amendment with long replacement clause text; clearly scoped to changing parts of an existing ordinance/code.
* amendment_targeted_text: explicit amendment with small, focused change (one/few sections or short clauses).
* map_rezoning_order: rezoning ordinance/order changing zoning district classification for specified parcel(s)/area(s). MUST be printed instrument text (not reported approvals).
* variance_special_use_order: printed order/decision/resolution granting/denying a variance, special exception, conditional/special use. MUST be printed instrument text (not reported approvals).
* procedural_notice_only: zoning hearing/adoption/procedural notice without printed clause-structured zoning instrument text.
* non_zoning_ordinance: legal/ordinance text present but not zoning-law text (or zoning only appears as a non-substantive cross-reference).
* uncertain: OCR noise or ambiguity prevents reliable determination.

Hard gating for "order" classes (prevents meeting-minutes false positives)

* map_rezoning_order and variance_special_use_order REQUIRE printed instrument-style language (ORDINANCE/ORDER/RESOLUTION/DECISION headings, "Be it ordained/enacted", "It is hereby ordered/resolved", sectioned clauses, "hereby rezoned", "variance ... hereby granted/denied", etc.).
* If the text is minutes/reporting (motions, seconds, carried; "approved a variation"; "board voted"), classify as zoning_discussion unless actual instrument clauses are printed.

C) Publication completeness (separate axis; keeps "full_issue" strict)

* none
* self_contained_in_issue: the issue contains start-of-document context AND adoption/completion context tied to the code/ordinance text (may be on different pages).
* continues_beyond_issue: ONLY if the issue explicitly says the text continues in another issue (e.g., "to be continued next week/next issue", "will be published in subsequent issues").

  * Do NOT treat phrases like "after three publications", "published for three successive weeks", or "published on [dates]" as continues_beyond_issue; those are publication/effectiveness mechanics, not cross-issue continuation of the printed text.
* unknown: you cannot find clear start/adoption/completion context anywhere in the issue.

D) Operativity (separate axis; explicit cues only)

* operative: explicit passage/adoption/approval/effective date/signatures/attestation tied to the ordinance/order text in this issue.
* proposed: explicit "proposed/draft/introduced/first reading/to be considered" or hearing on proposed ordinance, without passage/adoption.
* unclear: mixed cues or missing adoption context.

Operativity clarification for notices (prevents "Published Sept 25" confusion)

* In a NOTICE, "Published [date]" usually means the notice publication date, not ordinance adoption or effective date.
* For zoning_notice/procedural_notice_only, set operativity=proposed unless the text explicitly says adopted/passed/approved/effective.

E) Legal action (separate axis; prefer unknown unless explicit)

* none
* adopt
* recodify_replace (explicit recodify/repeal-and-reenact/replace-in-its-entirety/codification/consolidated rewrite)
* amend
* repeal
* rezone
* grant_variance_special_use
* procedural_only (ONLY for procedural_notice_only)
* unknown

Full comprehensive code standard (for code_publication_full_issue; strict)
Use code_publication_full_issue only if ALL are true:

1. zoning_legal_text is present and is jurisdiction-wide code-scale zoning (not a single parcel order; not mostly notice logistics; not a news summary).
2. Ordinance/code structure is printed (articles/sections/definitions/clauses).
3. At least 3 of these 4 components are explicitly visible in printed clauses:

   * district framework/classifications
   * use rules (permitted/prohibited/conditional uses)
   * dimensional standards (setbacks/lot size/height/density/parking)
   * procedures/enforcement/administration (boards, permits, appeals, penalties, definitions)
4. Explicit adoption or recodification context is present in the issue and tied to that code text.
5. publication_completeness = self_contained_in_issue.
   If any condition is missing, do NOT use code_publication_full_issue; use code_publication_excerpt_or_installment and add quality_flag insufficient_text_for_confident_full_code if the text looks comprehensive but completeness/adoption tie is not proven.

code_components_present rules (prevents inference from summaries)

* If zoning_presence.primary_class is zoning_notice OR zoning_discussion OR unrelated: ALL four booleans must be false.
* If zoning_presence.primary_class is zoning_legal_text: set a component true ONLY if printed clauses in the issue show it.

Scope normalization (no ad-hoc values)
Allowed scope values only:

* none|citywide|district_wide|parcel_specific|mixed|unknown
  Interpretation:
* citywide means "jurisdiction-wide" (applies broadly across the governing jurisdiction, even if the jurisdiction is a county/township/village).
  Do not output other values.

Jurisdiction extraction + state_abbr normalization

* Extract ordinance_jurisdiction ONLY if the ordinance/order text explicitly names the governing authority (e.g., "City of ___", "Town of ___", "Village of ___", "County of ___", "State of ___") in an enacting header, authority line, or "Be it ordained..." clause.
* state_abbr must be a 2-letter uppercase postal abbreviation (e.g., NY, OH, UT). If the text gives "N. Y." or "Ohio" or "Utah", normalize to NY / OH / UT. If not sure, leave state_abbr empty.
* If multiple distinct governing bodies appear and you cannot confidently isolate which instrument you are classifying, set jurisdiction unknown and add quality_flag ambiguous_scope.
