DEFINITIONS (apply literally):
- Zoning anchor: explicit zoning references such as zoning, zoning ordinance/code, rezoning/rezone, zoning map, zoning district change (e.g., R-1 to C-1), Board of Adjustment/Zoning Board, Planning and Zoning Commission, variance, conditional/special use, planned development (PUD), overlay district.
- Printed zoning legal text (instrument text): the issue prints actual ordinance/code/decision language as clauses (sections/articles, amendments "is hereby amended", "be it ordained/enacted", "shall", definitions, enforcement, etc.). This is NOT satisfied by descriptive prose summarizing an ordinance, captions/titles only, or an agenda/docket list.
- Zoning notice (no printed instrument text): hearing logistics, "notice is hereby given", agenda items, "first reading", "introduced", inspection/copy availability, publication logistics, ordinance captions without sectioned clauses.

STRICT PRINTED-LAW GATE (hard requirement for zoning_legal_text):
You may set zoning_presence.primary_class = zoning_legal_text ONLY if BOTH are true:
A) There is a zoning anchor.
B) You can quote a strong-form clause marker from printed instrument text (not narrative reporting, not a caption-only line, not a docket list).

Strong-form clause markers that DO qualify (any one is sufficient if clearly part of clause text):
1) Enacting / ordaining language: BE IT ORDAINED, BE IT ENACTED, DOES ORDAIN, HEREBY ORDAINS/ENACTS, NOW THEREFORE + ordain/enact, IT IS HEREBY ORDAINED/ENACTED.
2) True section/article clause structure: SECTION 1. / Sec. 1. / SECTION 2. etc with clause text, or ARTICLE headings with multiple sections/clauses AND normative language (shall, prohibited, permitted, amended, repealed).
3) Amendment clause text (allowed even inside a notice): phrases like Amend Section [number] to provide that..., Section [number] is hereby amended..., amended to read as follows, repealed and reenacted, insert/delete/strike/add language tied to a section.
4) Formal decision/order instrument (variance/special use): NOTICE OF DECISION / ORDER where a Board of Adjustment/Zoning Board grants/denies a variance/special exception/conditional use and includes official appeal/procedure language (not meeting-minutes reporting).

Hard blockers (these force NOT zoning_legal_text; classify as zoning_notice or zoning_discussion instead):
- Agenda/docket summary veto (Brownsville rule): lines like Ord. No. ___: To Rezone From ___ to ___, or ordinance-number listings, or "Passage at first reading" lists, WITHOUT any of the strong-form clause markers above are NOT printed law text. Treat as zoning_notice + procedural_notice_only. (This is a specific policy target.)
- Caption-only veto: AN ORDINANCE ... / ORDINANCE NO. ... titles or captions without sectioned clauses or enacting language are NOT printed law text.
- Narrative/reporting veto: sentences like "an ordinance was introduced/approved/will go into effect" or "general description of zone classifications follows" are NOT printed law text even if they mention sections/articles in prose. (Example pattern: "An addition to Article X, section 10.7 ..." in a news story is still narrative unless clause text is printed.)
- Non-zoning ordinance veto: If the only strong-form ordinance text is clearly non-zoning (traffic/health/bonds/etc.) and zoning material is only notice/discussion, do NOT label zoning_legal_text. You may set legal_document.class = non_zoning_ordinance while keeping zoning_presence as zoning_notice or zoning_discussion.

Class definitions:
- code_publication_full_issue: self-contained publication of a comprehensive zoning code in this issue. Must include adoption/recodification context AND broad code text. Must show at least 3 of 4 components (district framework, use rules, dimensional standards, procedures/enforcement/admin) in printed clauses. Be conservative.
- code_publication_excerpt_or_installment: code-like zoning text is printed but not proven complete/self-contained in this issue, or explicitly indicates publication beyond this issue. NOTE: within-issue "Continued from Page X" is NOT by itself cross-issue; only explicit "continued next issue/week" or "published in subsequent issues" indicates beyond-issue installment.
- amendment_substantial_text: amendment/revision prints long replacement clause text but is clearly limited vs a full code publication.
- amendment_targeted_text: short, targeted amendment clauses (few sections/lines).
- zoning_ordinance_limited_scope: printed ordinance/regulations for a specific overlay/PUD/planned development/special district with detailed standards/conditions; more than a simple rezoning label change.
- map_rezoning_order: printed ordinance/order that rezones specific parcels/areas (district change like R-1 to C-1) AND passes the printed-law gate. Do NOT use for docket summaries.
- variance_special_use_order: formal board decision/order granting/denying a variance/special exception/conditional use that passes the printed-law gate (not meeting minutes).
- procedural_notice_only: zoning notices/hearings/inspection availability/agenda listings; no printed clause text.
- non_zoning_ordinance: printed ordinance clause text exists but is not zoning; zoning material (if any) is notice/discussion only.
- none: no zoning legal document and no non-zoning ordinance text worth labeling.
- uncertain: only when OCR/noise is so severe you cannot decide zoning presence/type even conservatively.

OPERATIVITY (separate axis; do not treat publication-date footers as adoption):
Allowed values: none|operative|proposed|unclear

- proposed cues: proposed, draft, introduced, first reading, to be considered, public hearing will be held, planning commission recommends, copy may be examined.
- IMPORTANT: A footer like Published July 22, 1998 or publish dates in a notice are publication logistics, NOT passage/effective by themselves.
- If mixed/missing: unclear.
