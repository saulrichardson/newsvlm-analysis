You are classifying one newspaper issue transcript that may contain:
- zoning law text,
- zoning hearing notices,
- zoning discussion without law text,
- non-zoning legal notices/ordinances, and
- unrelated content.

Important scope:
- Classify conservatively.
- "full code" is rare and should only be used when the text itself shows broad, standing zoning code structure.
- If uncertain between full code vs amendment/notice/discussion, do NOT choose full code.

Primary objective:
1) Detect whether any zoning content is present.
2) If present, classify what kind of zoning material it is.
3) Distinguish comprehensive full zoning code from narrower amendments/notices.

Definitions:
- zoning anchor: explicit references such as "zoning", "zoning ordinance", "rezoning", "zoning map", zoning district changes (e.g., R-1 to C-1), zoning board/commission, variance, conditional/special use.
- zoning legal text: operative or proposed legal language that actually enacts/changing zoning rules/order.
- hearing notice: formal notice of a zoning hearing with schedule/place context.
- zoning discussion: zoning is discussed/reported but legal text is not printed.

Full-code standard (strict):
Use full_code_comprehensive only when ALL are true:
- The issue appears to print standing citywide zoning code text (not only one parcel/district action), and
- At least 3 of these 4 components are clearly present in printed legal text:
  1. district framework/classifications,
  2. use rules (permitted/prohibited/conditional uses),
  3. dimensional standards (lot size, setbacks, height, density, parking),
  4. procedures/enforcement/administration (boards, permits, appeals, penalties, definitions), and
- There is explicit adoption/recodification context (for example: ordinance adoption/enactment language for the zoning code, code replacement/recodification language, effective-date enactment context tied to the code text).

If fewer than 3 components are visible, do not assign full_code_comprehensive.
If components are broad but explicit adoption/recodification context is missing, use comprehensive_code_excerpt (not full_code_comprehensive).
